Module OWLDatatypes

Most of the OWL 2 datatypes are handled directly by RDFLib. However, in some cases, that is not good enough. There are two major reasons for this

  1. Some datatypes are missing from RDFLib and required by OWL 2 RL
  2. In other cases, though the datatype is present, RDFLib is fairly lax in checking the lexical value of those datatypes. Typical case is boolean.

Some of these defficiencies are handled by this module. All the functions convert the lexical value into a python datatype (including, in some cases, returning the original string) which will be used, eg, for comparisons (equalities). If the lexical value constraints are not met, exceptions are raised (to be handled by other places to, eventually, generate error triplets into the final output).

Requires: RDFLib, 2.2.2. and higher

_strToAnyURI(v)

 

Rudimentary test for the AnyURI value. If it is a relative URI, then some tests are done to filter out mistakes. I am not sure this is the full implementation of the RFC, though, may have to be checked at some point later.

Parameters:
  • v - the literal string defined as a URI @return the incoming value
Raises:
  • ValueError - invalid URI value

_strToBase64Binary(v)

 

Rudimentary test for the base64Binary value. The problem is that the built-in b64 module functions ignore the fact that only a certainl family of characters are allowed to appear in the lexical value, so this is checked first.

Parameters:
  • v - the literal string defined as a base64encoded string @return the decoded (binary) content
Raises:
  • ValueError - invalid base 64 binary value

_strToBool(v)

 

The built-in conversion to boolean is way too lax. The xsd specification requires that only true, false, 1 or 0 should be used...

Parameters:
  • v - the literal string defined as boolean @return corresponding boolean value
Raises:
  • ValueError - invalid boolean values

Test (and convert) a generic numerical type, with a check against a lower and upper limit.

  • interval - lower and upper bounds (non inclusive). If the value is None, no comparison should be done
  • conversion - conversion function, ie, int, long, etc

_strToDateTimeAndStamp(incoming_v, timezone_required=False)

 

Test (and convert) datetime and datetimestamp values.

Parameters:
  • incoming_v - the literal string defined as a hexa number
  • timezone_required - whether the timezone is required (ie, for datetimestamp) or not @return datetime
Raises:
  • ValueError - invalid datetime or datetimestamp

_strToDecimal(v)

 

The built in datatype handling for RDFLib maps a decimal number to float, but the python version 2.4 and upwards also has a Decimal number. Better make use of that to use very high numbers. However, there is also a big difference between Python's decimal and XSD's decimal, because the latter does not allow for an exponential normal form (why???). This must be filtered out.

Parameters:
  • v - the literal string defined as decimal @return Decimal if the local python version is >= 2.4, float otherwise
Raises:
  • ValueError - invalid decimal value

_strToDouble(v)

 

Test and convert a double value into a Decimal or float. Raises an exception if the number is outside the permitted range, ie, 1.0E+310 and 1.0E-330. To be on the safe side (python does not have double!) Decimals are used if possible. Upper and lower values, as required by xsd, are checked.

Parameters:
  • v - the literal string defined as a double @return Decimal if the local python version is >= 2.4, float otherwise
Raises:
  • ValueError - invalid value

_strToFloat(v)

 

Test and convert a float value into Decimal or (python) float. Raises an exception if the number is outside the permitted range, ie, 1.0E+40 and 1.0E-50.

Parameters:
  • v - the literal string defined as a float @return Decimal if the local python version is >= 2.4, float otherwise
Raises:
  • ValueError - invalid value

use_OWL_lexical_conversions()

 

Registering the datatypes item for RDFLib, ie, bind the dictionary values. The 'bind' method of RDFLib adds extra datatypes to the registered ones in RDFLib, though the table used here (ie, _OWLDTToPYTHON) actually overrides all of the default conversion routines.
